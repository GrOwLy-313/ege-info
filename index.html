01 Информационные модели и графы БАЗОВЫЙ · 1 балл · ~3 мин
Граф
Состоит из вершин (пункты) и рёбер (дороги). Рёбра — направленные (стрелка) или ненаправленные (линия).
Степень вершины
Кол-во рёбер из вершины. Главный инструмент — считай степени для сопоставления с таблицей.
Матрица смежности
Таблица со звёздочками. Кол-во звёздочек в строке = степень вершины.
Весовая матрица
Таблица с числами — длина дороги. Пусто = дороги нет.
3 типа задания
Тип 1 — матрица смежности: найти номера вершин САМЫЙ ПРОСТОЙ
Алгоритм:
1Посчитай степень каждой вершины на графе
2Посчитай звёздочки в каждой строке таблицы
3Уникальная степень → однозначное сопоставление
4Зная соседей найденных — ищи остальные
5Симметричный граф → два номера взаимозаменяемы
Пример разбора:
Dстепень 5 — уникальная → номер 7
Aстепень 3 — уникальная → номер 3
B,E→ остаются номера 2 и 6
Ответ: 26
Тип 2 — весовая матрица: найти сумму дорог СРЕДНИЙ
1Расставь степени → подпиши буквы к номерам
2Найди пересечение нужных строк и столбцов в весовой матрице
3Сложи найденные числа → ответ
Тип 3 — весовая матрица: кратчайший путь РЕДКИЙ
1Выполни анализ как в типе 2
2Перебери все пути из A в C, посчитай длину каждого
!Меньше рёбер ≠ короче путь! Проверяй все варианты.
3Выбери минимальное значение
Python-помощник
Шаблон перебора (типы 1 и 2) — для проверки, не замена анализу
from itertools import permutations

# Все рёбра графа
graph = 'EF EC EB BC BD DA AC AF'.split()

# Для строки 1,2,3... — номера столбцов со звёздочкой
matrix = '36 456 145 236 23 124'.split()

print(*range(1, len(matrix) + 1))
for i in permutations('ABCDEF'):
    if all(str(i.index(x)+1) in matrix[i.index(y)] for x,y in graph):
        print(*i)
Как заполнить: в graph — рёбра через пробел; в matrix — номера столбцов со звёздочкой для каждой строки. Вывод: строка с номерами, под ней — буквы.

Полный перебор — когда есть незаполненные ячейки в таблице условия
from itertools import permutations, product

def F(x, y, z, w):
    return ((w <= y) <= x) or (not(z)) # ← вставь свою функцию

for a in product([0,1], repeat=7): # repeat = кол-во незаполненных ячеек
    t = [(a[0], a[1], 1, a[2]), # строка 1 (известное=число, неизвестное=a[i])
         (a[3], 0, a[4], a[5]), # строка 2
         (a[6], 1, 0, 0)] # строка 3
    if len(set(t)) != 3: continue # строки должны быть разными
    for i in permutations("xyzw"):
        if [F(**dict(zip(i,s))) for s in t] == [0, 0, 0]: # [1,1,1] если F=1
            print(i)
Как заполнить: вставь функцию в def F; в t — строки из таблицы условия; repeat= = кол-во незаполненных ячеек.

04 Кодирование и декодирование БАЗОВЫЙ · 1 балл
Условие Фано
Ни одно кодовое слово не является началом другого. Если буква стоит в узле — все её потомки в дереве недоступны.
Двоичное дерево
Влево — добавляем 0, вправо — 1. Код буквы — путь от корня до листа.
Правило недоступности
Занят узел 10 → недоступны 100, 101, 1000... Занят лист 101 → нельзя ставить букву на 10.
2 типа задания
Тип 1 — найти кратчайший код буквы
1Построить двоичное дерево нужной глубины
2Отметить все известные коды букв
3Отметить недоступные узлы (потомки и предки занятых)
4Найти кратчайший свободный лист
5Если несколько одинаковых — взять с наименьшим/наибольшим значением (по условию)
Тип 2 — найти длину закодированного слова
1Подсчитай частоту каждой буквы в слове
2Найди свободные листы (шаги 1–3 типа 1)
3Самому частому — кратчайший код, далее по убыванию частоты
4Ответ = сумма (длина кода × кол-во вхождений)
Если два варианта кажутся равными — считай оба. Пример: «Е» (3×) с кодом 2 символа + «Р» (2×) с кодом 4 = 2×3+4×2=14, а оба по 3 символа = 3×3+3×2=15. Выгоднее первый вариант.

К: 2 × 2 = 4
О: 3 × 1 = 3
Л: 2 × 3 = 6

05 Простые алгоритмы и системы счисления БАЗОВЫЙ · 1 балл
Дан алгоритм: перевести число в систему счисления → обработать по правилу → перевести обратно. Нужно найти N, при котором результат удовлетворяет условию. Решается кодом на Python.
Основной шаблон — dwa(N) и R(N)
Основной шаблон
# Функция перевода в двоичную:
def dwa(N):
    b = ''
    while N > 0:
        b = str(N % 2) + b
        N //= 2
    return b

# Функция алгоритма (R — результат)
def R(N):
    b = dwa(N)
    if sum(list(map(int, b))) % 3 == 0: # сумма цифр делится на 3
        b += b[-2:] # дописать последние 2 цифры
    else:
        b += dwa((sum(list(map(int, b))) % 3) * 3)
    return int(b, 2) # вернуть как десятичное

print(min(N for N in range(1, 1000) if R(N) >= 300))
Что меняется в каждом задании: условие внутри R(N) и что ищем (min/max, условие на R).

    alph = digits + ascii_uppercase # '0123456789ABC...Z'
    result = ''
    while num:
        result += alph[num % base]
        num //= base
    return result[::-1] # развернуть строку

    while num:

Объединение и пересечение — в чём разница
Объединение
Всё, что покрывает хотя бы одна из фигур. Если фигуры частично перекрываются — общая область считается один раз.
Формула для точек:
Точек объединения = Фигура 1 + Фигура 2 − Пересечение
Пересечение
Только то, что покрывают обе фигуры одновременно. Это область, где они накладываются друг на друга.
Стороны пересечения:
= сторона фигуры − смещение второй фигуры
Что именно считать — площадь, периметр или точки
Разница между тремя типами вопросов
Площадь
Считаем в клетках — стороны в отрезках (не точках).
Прямоугольник 7×12:
S = 7 × 12 = 84
Стороны берём прямо из алгоритма (fd(N)).
Периметр
Тоже в отрезках, обходим по контуру.
Прямоугольник 7×12:
P = 2 × (7 + 12) = 38
Для пересечения — стороны пересечения.
Точки
Точки на узлах сетки — включая границы.
Прямоугольник 7×12:
(7+1) × (12+1) = 104
+1 к каждой стороне, т.к. точек на отрезке длиной N всегда N+1.
Не включая точки на линиях (только внутри): считать стороны − 1, т.е. (N−1) × (M−1). Если это прямоугольник 7×12 → 6 × 11 = 66 внутренних точек.
Площадь пересечения — как считать
Считаем сколько раз черепаха прошла каждую сторону
Если две фигуры пересекаются, смотришь: на сколько клеток они перекрываются по горизонтали и на сколько по вертикали. Это и есть стороны пересечения.
Пример:
Фигура 1: стороны 16 и 22
Смещение второй фигуры: 5 вперёд и 5 вправо
→ Пересечение: (16−5) × (22−5) = 11 × 17 = 187
Перекрытие по горизонтали = 16 − 5 = 11
Перекрытие по вертикали = 22 − 5 = 17
Для точек пересечения:
(11 + 1) × (17 + 1) = 12 × 18 = 216
+1 к каждой стороне
Объединение точек:
Фигура 1: (16+1)×(22+1) = 17×23 = 391
Фигура 2: предположим 200 точек
Итого: 391 + 200 − 216 = 375

Формулы
Изображение
V = h × w × i
V — объём в битах
h — высота в пикселях
w — ширина в пикселях
i — глубина цвета (бит на пиксель)
N = 2ⁱ     i = log₂(N)
N — количество цветов в палитре
i — глубина цвета (бит на пиксель)
Звук
V = k × t × i × η
V — объём в битах
k — количество каналов: моно = 1, стерео = 2, квадро = 4
t — длительность в секундах
i — глубина кодирования (бит, она же разрешение)
η — частота дискретизации в Гц
Передача данных
V = q × t
V — объём данных в битах
q — скорость канала в бит/с
t — время передачи в секундах
Видео
V = h × w × i × fps × t
h, w — высота и ширина кадра в пикселях
i — глубина цвета (бит на пиксель)
fps — частота кадров (кадров в секунду)
t — длительность в секундах

Всё считаем в битах, в конце переводим в нужные единицы
1 байт
1 Кбайт
1 Мбайт
1 Гбайт
1 кГц
= 8 бит = 2³ бит

Что бывает в задачах
Сжатие
«сжат на X%» → сжатый = оригинал × (1 − X/100)
Пример: сжат на 40% → сжатый = оригинал × 0.6
«оригинал больше сжатого на X%» → оригинал = сжатый × (1 + X/100)
«объём уменьшился в K раз» → сжатый = оригинал / K
Заголовки треков
Общий объём альбома = объём музыки + заголовки всех треков
Объём музыки = k × t_суммарное × i × η
Объём заголовков = N_треков × размер_заголовка
Передача в реальном времени
Объём за 1 секунду не должен превышать скорость канала.
V за 1с = k × i × η ≤ скорость канала
Из этого выражаем нужное — обычно i (глубину), округляем вниз.
Карты памяти
Снимков на одну карту = вместимость карты ÷ объём одного снимка (вниз)
Снимков на последней карте = общее_кол-во % снимков_на_карту
Если остаток = 0, последняя карта заполнена полностью.
Изменение параметров
Если нужно найти новый объём — просто умножай/делй исходный на соотношение параметров.
моно → стерео: × 2  |  стерео → моно: ÷ 2  |  квадро → стерео: ÷ 2
Частота ×3.5: × 3.5  |  Разрешение ÷2.5: ÷ 2.5
Максимум / минимум цветов
Максимум цветов (объём ≤ X): i = X ÷ (h×w), округлить вниз, N = 2ⁱ
Минимум цветов (объём > X): i = X ÷ (h×w), округлить вверх, N = 2ⁱ

Как устроен код
Три переменные — всегда одно и то же
ceil(log2(N))
ceil(i * bit / 8)
835 * byte
bit — сколько бит нужно на один символ. N = мощность алфавита (сумма всех символов)
byte — сколько байт занимает один номер длиной i символов

Тип 1 — найти длину номера
Известны: мощность алфавита и объём памяти. Найти: длину

Перебираем i — длину номера. Как только объём базы превысил порог — это и есть минимальная длина.

Известны: длина номера и мощность алфавита. Найти: объём памяти

Перебираем i — теперь это объём памяти в Мбайт. Ищем минимальное i, при котором всё помещается.
Сравнение трёх типов
Что меняется, а что остаётся
Тип 1 — длина
Перебираем: i = длину
Фиксировано: N (мощность)
Условие: объём базы > порога
Тип 2 — мощность
Перебираем: i = мощность
Фиксировано: длина
Условие: объём базы >= порога
Тип 3 — объём
Перебираем: i = Мбайт
Фиксировано: длина + N
Условие: объём базы <= i × 2²⁰
Знак условия зависит от формулировки: «требуется более X» → >, «не менее X» → >=, «не превышает X» → <=. Всегда смотри на точную формулировку.

13 IP-адреса и маски подсети ПОВЫШЕННЫЙ · 1 балл
Работаем с IP-адресами, масками и сетями. Все задания решаются через модуль ipaddress в Python. Нужно перебирать адреса или маски и проверять условие.
Ключевые понятия
IP-адрес (IPv4)
32-битное число, 4 октета: 192.168.1.3. Состоит из адреса сети + адреса узла.
Маска подсети
Биты 1 = часть сети, биты 0 = часть узла. Единицы всегда слева непрерывно. Префикс = кол-во единиц.
Адрес сети
IP ∧ Маска = адрес сети. Все биты узла = 0. Первый адрес сети, нельзя назначить устройству.
Широковещательный
Все биты узла = 1. Последний адрес сети, тоже нельзя назначить устройству.
Кол-во адресов
N = 2^(32−префикс). Назначить устройствам можно N−2 (минус сеть и широковещательный).
CIDR-нотация
192.168.0.0/24 — префикс 24 = маска 255.255.255.0. /16 = 255.255.0.0. /8 = 255.0.0.0.
